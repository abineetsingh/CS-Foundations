Prefix Sums: Precompute, Then Answer Instantly

Today's skill: answer "sum of any range" queries in O(1) by paying O(n) once, up front.

Why this, now

Prefix sums introduce the third great cost-shifting move (after Lesson 0002's space-for-time and Lesson 0007's update-don't-recompute): preprocessing. Spend once, answer many times cheaply — the shape of every index, cache, and database in existence.

The construction

Define prefix[i] = sum of the first i elements (with prefix[0] = 0):

arr    =      [3,  1,  4,  1,  5,  9]
prefix = [0,   3,  4,  8,  9, 14, 23]

def build_prefix(arr):
    prefix = [0]
    for x in arr:
        prefix.append(prefix[-1] + x)   # each entry: previous + one element
    return prefix

Then any range sum falls out by subtraction — everything before j, minus everything before i:

def range_sum(prefix, i, j):        # sum of arr[i..j] inclusive
    return prefix[j + 1] - prefix[i]   # O(1)

sum(arr[2..4]) = prefix[5] − prefix[2] = 14 − 4 = 10 = 4+1+5. ✓

Costs: build O(n) time and O(n) space, then every query O(1). One build amortizes over any number of queries. Contrast: answering q queries by slicing costs O(q·n).

The off-by-one armor

The leading 0 and length n+1 are not style — they eliminate special-casing ranges that start at index 0. Adopt this convention permanently: prefix has n+1 entries, prefix[0] = 0, range [i..j] = prefix[j+1] − prefix[i]. Interviewers watch for off-by-ones here more than anywhere else.

Level 2: prefix sums meet the hash map

Count subarrays summing exactly to k (a famous medium). Insight: a subarray [i..j] sums to k exactly when prefix[j+1] − prefix[i] = k — i.e., while scanning, we want to know: how many earlier prefixes equal current − k? That's a "remember the past in a dict" question (Lesson 0005):

from collections import defaultdict

def count_subarrays_sum_k(arr, k):
    counts = defaultdict(int)
    counts[0] = 1                     # empty prefix
    running, result = 0, 0
    for x in arr:
        running += x
        result += counts[running - k]   # earlier prefixes that complete a k-sum
        counts[running] += 1
    return result

O(n) time for a problem that looks like it needs all O(n²) subarrays. Patterns compose — this is prefix sums × hash map, and it's the level the NeetCode 150 expects.

Mission link · ML/AI

To sample a token from a probability distribution, frameworks build the cumulative distribution — a prefix sum of probabilities — then find where a random number lands (with binary search, Lesson 0014: another composition). Cumulative ops are so central that GPUs have a parallel primitive for them (cumsum/scan).
