Linked Lists I: Nodes, Traversal, and the Dummy Trick

Today's skill: build and walk a chain of nodes confidently — pointers instead of indices.

Why this, now

Linked lists are the simplest pointer-based structure — the training ground for the reference-juggling that trees and graphs demand. Interviews use them to check whether you can manipulate references without losing nodes. They're also half of the LRU cache (Lesson 43), a genuine systems structure.

The structure

No contiguous block. Each element is a free-floating node that knows its value and where the next node lives:

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

# 3 → 1 → 4 → None
head = ListNode(3, ListNode(1, ListNode(4)))

You hold only head. Everything else is reachable by following .next. The costs invert Lesson 03's array table:

OperationArray (list)Linked list
Access by indexO(1)O(n) — must walk
Insert/delete at frontO(n)O(1)
Insert/delete after a node you holdO(n)O(1) — rewire two pointers
Find a valueO(n)O(n)
Memory layoutcontiguous, cache-friendlyscattered, pointer-chasing

The two core moves

Traversal — the while loop you'll write 100 times

current = head
while current:                 # runs until current is None
    print(current.val)
    current = current.next     # THE step: move the pointer, don't index

Insertion/deletion — rewiring

# insert new_node AFTER node:            # delete the node AFTER node:
new_node.next = node.next                 # node.next = node.next.next
node.next = new_node

Order matters: in insertion, point the new node out before pointing anything at it, or the rest of the chain is orphaned. Losing the chain is the classic linked-list bug.

The dummy-node trick

Deleting the head is a special case (nothing points at it). Kill the special case by planting a fake node before the real head:

def delete_all(head, val):
    dummy = ListNode(0, head)         # sits before the real head
    node = dummy
    while node.next:
        if node.next.val == val:
            node.next = node.next.next   # unlink — works even for the head
        else:
            node = node.next
    return dummy.next                  # the (possibly new) real head

Like prefix[0] = 0 in Lesson 09, the dummy is off-by-one armor — one line that deletes an entire class of edge-case bugs. Use it whenever the head might change.

Mission link · ML/AI

The array-vs-linked-list trade-off — contiguity is fast to scan, pointers are fast to restructure — is a hardware truth that shapes ML systems: tensors are contiguous precisely because GPUs hate pointer-chasing. When you meet PagedAttention (vLLM), you'll recognize it instantly: KV-cache memory managed in linked blocks, accepting pointer indirection to avoid O(n) reallocation. Same trade, billion-dollar setting.
