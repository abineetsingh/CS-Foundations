Hash Maps & Sets: The Interview Workhorse

Today's skill: use dicts and sets to turn O(n²) solutions into O(n) — and explain why the lookup is O(1).

Why this, now

If you learn one data structure deeply, make it this one. "Use a hash map" is the correct optimization in more interview problems than every other structure combined. Two Sum — historically the most-asked interview question on earth — is a hash map in disguise.

How hashing works (enough to explain in an interview)

A hash table is just an array (Lesson 0003) used cleverly:

  1. A hash function turns any key into a number: hash("cat") → 8_764_213…
  2. Modulo the array size gives a slot: 8_764_213 % 8 → slot 5. The key/value goes in slot 5.
  3. Lookup replays the same arithmetic — compute, jump, done. No searching → O(1) average.

Two keys can land in the same slot (a collision); the table handles it by chaining or probing, and resizes when it gets crowded. If everything collided, lookups would degrade to O(n) — that's why we say O(1) average (Python Wiki). In interviews, average case is the expected answer; mentioning the caveat once is a plus.

This also explains Lesson 0004's cliffhanger: a key's hash must never change, so keys must be immutable — strings, numbers, tuples yes; lists, dicts, sets no.

The signature move: trade space for a complexity class

Two Sum: find two numbers in nums adding to target. Brute force checks all pairs — O(n²). The hash-map insight: for each number, the partner I need is target − num. Have I seen it?

def two_sum(nums, target):
    seen = {}                        # value -> index
    for i, num in enumerate(nums):
        need = target - num
        if need in seen:             # O(1) average
            return [seen[need], i]
        seen[num] = i
    return []

One pass, O(1) work per element: O(n) time, O(n) space. Note the pattern — don't search for the partner (O(n) each time); remember everything you've walked past and ask in O(1). This "remember the past in a dict" move solves dozens of problems.

Counting: Valid Anagram

from collections import Counter

def is_anagram(s, t):
    return Counter(s) == Counter(t)   # O(n) build, O(1)-ish compare (≤26 letters)
