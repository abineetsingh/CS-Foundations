Latency Numbers

The dozen numbers behind every architecture argument. Orders of magnitude — memorize ratios, not digits. Print me.

OperationTimeAnchor
L1 cache reference~1 nsMemory: nanoseconds
Main memory (RAM) reference~100 ns
Read 1 MB sequentially from RAM~10 µs
SSD random read~100 µsSSD: microseconds
Read 1 MB from SSD~1 ms
Round trip inside one datacenter~0.5 msNetwork: milliseconds
Spinning-disk seek~10 ms
Round trip, same continent~30–70 ms
Round trip, opposite side of Earth~150–300 ms

Rules that fall out of the table

  • Each tier ≈ 1000× the one above. RAM → SSD → network.
  • Latency = sum of sequential hops. Count the round trips; the slowest hop dominates.
  • N+1 queries: N sequential DB calls ≈ N × 1.5 ms. Batch or join instead.
  • Can't beat lightspeed → move data closer: the entire justification for caches and CDNs.
  • TLS+TCP setup costs extra round trips before the first byte of your request — why connections are reused (keep-alive, pools).

Estimation crib

  • 1 day ≈ 86,400 s ≈ 10⁵ s · 1 month ≈ 2.6 × 10⁶ s · 1 year ≈ 3.15 × 10⁷ s
  • Requests: avg QPS = daily actions / 10⁵; peak ≈ 2–10× average
  • A well-tuned single Postgres box: ~thousands of simple QPS; a Redis box: ~100k ops/s — order-of-magnitude anchors only
