# Mission: Application Security & Security Engineering

## Why
Build the fundamentals needed to pass application-security and security-engineering interviews, then turn those techniques into the ability to design, build, review, and test production software securely. Learn common vulnerabilities from both sides—reproduce them safely in controlled labs, repair their root causes, and prevent them through security-by-design practices.

## Success looks like
- Explain and demonstrate the root cause, exploit path, impact, remediation, and durable prevention for common web and API vulnerabilities
- Threat-model an unfamiliar production system, identify its trust boundaries and highest-value abuse cases, and turn them into concrete security requirements and tests
- Review Python and JavaScript code for security flaws, validate findings without harming real systems, and communicate risk with evidence and practical fixes
- Design stack-agnostic application and service architectures with sound identity, authorization, cryptography, secrets, data protection, container, cloud, and IAM controls
- Threat-model and harden production AI, RAG, and agent systems across prompts, context, models, tools, data, supply chain, providers, infrastructure, and security evaluations
- Drive a security-engineering interview: clarify assets and adversaries, reason about tradeoffs, investigate a vulnerable design or code sample, and propose layered mitigations

## Constraints
- Security beginner with Python, JavaScript, and web fundamentals
- 1–2 hours per day; short lessons plus safe hands-on labs, retrieval practice, and periodic design/code-review drills
- Production principles should transfer across web, mobile-backed, API, service, and cloud application stacks
- Offensive techniques are practiced only against deliberately vulnerable local or hosted training targets with explicit authorization

## Out of scope
- Unauthorized testing, stealth, persistence, malware development, credential theft, or exploitation of real third-party systems
- Incident-response runbooks and organization-specific operational playbooks
- Compliance certification study and vendor-specific cloud certification trivia
- Deep cryptographic protocol design or inventing custom cryptography; use vetted constructions and maintained libraries
- Frontier model training research and novel adversarial-ML algorithms; the course secures production systems that consume or operate models
