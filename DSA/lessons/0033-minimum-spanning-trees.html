Minimum Spanning Trees

Today's skill: connect all nodes at minimum total cost — Kruskal's algorithm, which is just sorting plus the Union-Find you already own.

Why this — and why optional

MST appears in AI-lab screens rarely; it's here because Kruskal's is a two-lesson combo (0016 sorting + 0031 Union-Find) you get nearly free, and "Min Cost to Connect All Points" does show up. If time is tight, read the Kruskal section, skip the rest, move to Phase 4.

The problem

Given a weighted undirected graph, choose edges connecting ALL nodes with minimum total weight. The answer is always a tree (V−1 edges): any cycle contains a removable edge that keeps everything connected but costs less — so optimal solutions never contain cycles.

Kruskal's: greedy + Union-Find

Sort edges cheapest-first; take each edge unless it would close a cycle — and "would close a cycle" is exactly Union-Find's union() == False (Lesson 0031):

def mst_cost(n, edges):                    # edges: (w, u, v)
    edges.sort()                           # cheapest first — O(E log E)
    uf = UnionFind(n)                      # Lesson 0031, verbatim
    total, used = 0, 0
    for w, u, v in edges:
        if uf.union(u, v):                 # False ⇔ u,v already connected ⇔ cycle
            total += w
            used += 1
            if used == n - 1:              # a spanning tree is complete
                break
    return total if used == n - 1 else -1  # -1: graph wasn't connected

O(E log E), dominated by the sort. Why is the greedy choice safe? The cut argument, one sentence: for any way of splitting the nodes in two, the cheapest edge crossing the split belongs to some MST — taking cheapest-first never burns you. (Compare Dijkstra's safety argument from Lesson 0032: both greedy proofs pivot on "nothing later can beat this." Greedy-with-proof is Phase 4's Lesson 0038 topic; you're previewing it.)

Prim's, for recognition

The alternative: grow one tree from a seed, always adding the cheapest edge leaving the tree — a min-heap of frontier edges. It's Dijkstra's twin with the heap keyed on edge weight instead of total path distance. If you know Dijkstra, you can reconstruct Prim's; most people implement Kruskal's (fewer moving parts) and mention Prim's.

Mission link · ML/AI

Single-linkage hierarchical clustering IS Kruskal's algorithm — merge the closest clusters repeatedly, and the merge sequence is exactly MST construction on the distance graph. Cut the MST's k−1 heaviest edges and you have k clusters. Data-center network design and minimum-cost cabling are the literal use case.
