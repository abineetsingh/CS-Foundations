Two Pointers

Today's skill: recognize when two indices walking toward each other can replace a nested loop — O(n²) → O(n).

Why this, now

This is your first named pattern — the recognition units your mission's "spot the pattern in minutes" goal is made of. Two pointers is the easiest to learn and among the most frequently asked.

The idea

Many pair-finding problems seem to need all pairs (O(n²)). But when the data is sorted (or symmetric, like a palindrome), one comparison tells you which pointer is safe to move — so each element is visited once:

left = 0
right = len(arr) - 1
while left < right:
    # examine arr[left] and arr[right]
    # move exactly one pointer inward, based on a comparison

Every iteration shrinks the gap by one → at most n iterations → O(n) time, O(1) space.

Worked example 1: Valid Palindrome

def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True

Compare the outsides, walk inward. (The interview version skips non-alphanumeric characters — same skeleton plus two inner while skips.)

Worked example 2: Two Sum on a SORTED array

def two_sum_sorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        total = nums[left] + nums[right]
        if total == target:
            return [left, right]
        elif total < target:
            left += 1        # need a bigger sum → only moving left up can help
        else:
            right -= 1       # need a smaller sum → only moving right down can help
    return []

The crucial sentence — say it in interviews: "Because the array is sorted, if the sum is too small, no pair using nums[left] can work, so I can discard it safely." Each step provably eliminates candidates. That's why sortedness is the precondition; on unsorted data, use the dict from Lesson 0005 instead.
