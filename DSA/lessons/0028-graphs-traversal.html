Graphs: Representation & Traversal

Today's skill: model relationships as an adjacency list, and traverse any graph with the DFS/BFS you already own — plus the one new ingredient: the visited set.

Why this, now

Graphs are the most general structure you'll learn — networks, dependencies, state spaces are all graphs. Trees were the training wheels: a tree is just a graph with no cycles and one parent each. Your DFS and BFS transfer nearly verbatim, which is why this lesson lands in days instead of weeks. For your long game: neural networks are literally computation graphs.

Vocabulary + representation, in one block

Vertices (nodes) connected by edges; edges may be directed (one-way: follows, dependencies) or undirected (friendships); a graph may contain cycles. The workhorse representation is the adjacency list — each node maps to its neighbors:

edges = [(0,1), (0,2), (1,2), (2,3)]        #   0 ── 1
                                            #   │  ╱
from collections import defaultdict         #   2 ──── 3
adj = defaultdict(list)
for u, v in edges:
    adj[u].append(v)
    adj[v].append(u)          # undirected: both ways (omit for directed)

# adj = {0: [1,2], 1: [0,2], 2: [0,1,3], 3: [2]}

Space O(V + E); iterating a node's neighbors costs only its degree. The alternative — an adjacency matrix (V×V grid) — answers "are u, v connected?" in O(1) but costs O(V²) space and O(V) to list neighbors; mention it, use it rarely. Interview problems hand you edge lists — building adj is always your first move.

Traversal = tree traversal + a visited set

Trees couldn't loop back on you; graphs can. Without protection, a cycle traps DFS forever. The fix is one set:

def dfs(node, adj, visited):
    if node in visited:            # the ONLY new line vs. tree DFS
        return
    visited.add(node)
    for neighbor in adj[node]:
        dfs(neighbor, adj, visited)

def bfs(start, adj):
    visited = {start}              # mark on ENQUEUE, not on dequeue (see quiz)
    q = deque([start])
    while q:
        node = q.popleft()
        for nb in adj[node]:
            if nb not in visited:
                visited.add(nb)
                q.append(nb)

Each node enters once, each edge is examined once (twice undirected): O(V + E) time — the graph-traversal complexity you'll state twenty times in interviews. Everything from Lessons 0019–0020 carries: DFS for "explore/exists/combine," BFS for "nearest/levels" (BFS visits nodes in increasing distance from start — on unweighted graphs, first arrival IS the shortest path; that becomes Lesson 0032's foundation).

The starter application: connected components

How many separate friend-groups in this network? Traverse from every unvisited node; each launch discovers one whole component:

def count_components(n, adj):
    visited = set()
    count = 0
    for node in range(n):          # every node gets a chance…
        if node not in visited:    # …but only unvisited ones start a flood
            dfs(node, adj, visited)
            count += 1             # one launch = one component
    return count

The loop-over-all-starts wrapper matters beyond this problem: graphs (unlike trees) have no root, so coverage requires trying every node. This wrapper reappears in every "for the whole graph" question.

Mission link · ML/AI

A neural network's forward pass is a directed graph of operations — PyTorch builds it dynamically as your code runs, then walks it backward for gradients. Knowledge graphs, GNNs passing messages along edges (Lesson 0022's aggregation, generalized), agent state-spaces where nodes are world-states and edges are actions: the graph abstraction is arguably the single most load-bearing one in AI. You now hold its two fundamental algorithms.
