Union-Find (Disjoint Set Union)

Today's skill: track "which group is this in?" under a stream of merges — in effectively constant time per operation.

Why this, now

DFS answered "are these connected?" for a fixed graph. Union-Find answers it while edges keep arriving — without re-traversing anything. It's short enough to memorize completely, it's the engine of Kruskal's MST (Lesson 0033), and "accounts merging"-style questions are common at exactly your target companies.

The idea: groups as trees, groups named by roots

Every element points toward a parent; follow the chain and you reach a root — the group's representative. Two elements are in the same group iff they share a root:

parent: [0, 0, 1, 3]        0        3       find(2): 2 → 1 → 0. root = 0
                           /                 find(3): 3.          root = 3
                          1                  same_group(2, 3)? 0 ≠ 3 → no
                         /
                        2                    union(2, 3): parent[3] = 0
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))     # everyone starts alone: own root
        self.rank = [0] * n              # tree-height upper bounds

    def find(self, x):                   # walk to the root, flattening as we go
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])   # PATH COMPRESSION
        return self.parent[x]

    def union(self, a, b):               # merge the groups; True if they were separate
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False                 # already together — a redundant edge!
        if self.rank[ra] < self.rank[rb]:
            ra, rb = rb, ra              # attach shorter under taller (UNION BY RANK)
        self.parent[rb] = ra
        if self.rank[ra] == self.rank[rb]:
            self.rank[ra] += 1
        return True

The two optimizations (and the famous complexity)

  • Path compression: during find, repoint every node on the walk directly at the root — the tree flattens itself with use.
  • Union by rank: always hang the shorter tree under the taller — trees stay shallow.

Together: amortized O(α(n)) per operation — the inverse Ackermann function, ≤ 4 for any n that fits in the universe. Say "effectively constant, inverse Ackermann" in interviews; deriving it is nobody's expectation. (Either optimization alone gives O(log n) — still fine.)

Using it

# Number of components after each edge (dynamic connectivity):
uf = UnionFind(n)
components = n
for u, v in edges:
    if uf.union(u, v):
        components -= 1                  # a true merge: two groups became one

# Cycle detection in an UNDIRECTED graph:
# an edge whose endpoints ALREADY share a root closes a cycle
if not uf.union(u, v):  print("cycle!")

Mission link · ML/AI

Agglomerative clustering — repeatedly merging the closest clusters — is Union-Find's exact access pattern, and image segmentation algorithms (Felzenszwalb) merge pixel regions with literal DSU. Dataset deduplication at LLM-pretraining scale (MinHash near-duplicate detection) unions documents into duplicate-clusters across billions of items — inverse Ackermann is why that's feasible.
