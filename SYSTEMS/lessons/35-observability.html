Observability

One skill, learned today: give a system eyes — logs, metrics, traces — define an SLO on top, and never trust an average again.

Why this matters

Everything in phase 4 assumed you could "watch error rates" (canaries), "notice queue depth" (autoscaling), "detect death" (health checks). Today builds the watching itself. In interviews, "how would you know it's broken?" separates people who've operated systems from people who've drawn them.

The three pillars

PillarWhatAnswersCost profile
Logstimestamped, structured events (JSON, not prose): {"event":"charge_failed","user":42,"amount":50}"what exactly happened to request X?"huge volume — sample or retain briefly
Metricscheap pre-aggregated numbers over time: request count, error rate, latency histogram, queue depth"how is the system doing right now / vs yesterday?"tiny, keep forever — dashboards & alerts live here
Tracesone request's journey stitched across services, timed per hop, joined by a propagated trace id"WHERE did these 3 seconds go?"sampled (e.g. 1%) — the microservices debugger (30's murder mystery, solved)

The modern standard for emitting all three is OpenTelemetry; Prometheus + Grafana is the classic metrics stack. Names to drop; concepts to own.

Percentiles: the lie of the average

A dashboard says "average latency: 45 ms." Meanwhile 1 in 100 users waits 8 seconds. Averages bury outliers; latency is a distribution, so operators speak in percentiles: p50 (median), p99 (the worst 1%), p99.9. Two reasons the tail IS the product: heavy users hit you most often, so they meet the p99 constantly; and a page fanning out to 10 backend calls (30) suffers the tail on some call with probability 1−0.99¹⁰ ≈ 10% — tail latencies compound under fan-out. Alert on p99, chart p50/p95/p99 together, and treat any average with suspicion [Google SRE: monitoring].

SLOs: turning metrics into promises

An SLI is a measured indicator ("fraction of requests under 200 ms, successful"). An SLO is the target you commit to: "99.9% of requests succeed under 200 ms, monthly." The genius part is the error budget: 100% − SLO = the failure you're allowed (0.1% ≈ 43 min/month). Budget healthy → ship fast, take risks (34 canaries burn budget!). Budget spent → freeze features, fix reliability. It converts the eternal speed-vs-safety argument into arithmetic both sides accept [Google SRE: SLOs]. Alert on budget burn rate ("at this error rate, the month's budget dies in 6 hours"), not on every blip — that's the difference between paging and spam.

Hands-on: instrument your API (10 min)

Add the metrics pillar to the notes API with a middleware — this is real instrumentation, not a toy:

import time, collections
from fastapi import FastAPI, Request

app = FastAPI()
metrics = {"requests": collections.Counter(), "latencies_ms": collections.deque(maxlen=10_000)}

@app.middleware("http")
async def observe(request: Request, call_next):
    t0 = time.monotonic()
    response = await call_next(request)
    ms = (time.monotonic() - t0) * 1000
    metrics["requests"][f"{request.method} {request.url.path} {response.status_code}"] += 1
    metrics["latencies_ms"].append(ms)
    return response

@app.get("/metrics")
def get_metrics():
    lat = sorted(metrics["latencies_ms"])
    pct = lambda p: lat[int(len(lat) * p)] if lat else 0
    return {"counts": dict(metrics["requests"]),
            "p50_ms": round(pct(0.50), 2), "p95_ms": round(pct(0.95), 2),
            "p99_ms": round(pct(0.99), 2)}

Fire 100 curls at your endpoints, then curl localhost:8000/metrics: live counts by status code and real percentiles. Add a time.sleep(2) to one handler and watch p99 scream while p50 sits innocent — the average's lie, demonstrated on your own service. (Production swaps this dict for a Prometheus client; the shape is identical.)

AI-lab tie-in

LLM serving adds its own SLIs to the classic set: time-to-first-token (the p99 users feel), tokens/second, GPU utilization, KV-cache hit rate — canaried per model release (34). And agent observability is traces wearing a new hat: a tool-call chain is a distributed trace, span by span; "where did this agent burn 30 seconds" is the exact question tracing answers. Same pillars, new nouns.
