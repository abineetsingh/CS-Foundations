Grid Graphs: Islands & Flood Fill

Today's skill: see a 2-D grid as a graph — no adjacency list needed — and run DFS/BFS over cells.

Why this, now

Grid problems are the most-asked graph questions in practice ("Number of Islands" is a top-5 interview problem, period). They're also gentle: the graph is implicit, so you practice pure traversal without modeling overhead.

The reframe: cells are nodes, adjacency is arithmetic

A grid IS a graph: each cell (r, c) is a node; its neighbors are up/down/left/right. No adj dict needed — neighbors are computed:

DIRS = [(-1, 0), (1, 0), (0, -1), (0, 1)]      # the four directions

for dr, dc in DIRS:
    nr, nc = r + dr, c + dc
    if 0 <= nr < rows and 0 <= nc < cols:      # bounds check = edge exists
        ...visit (nr, nc)...

Memorize this idiom — the DIRS list and the bounds check appear in every grid problem. (Diagonals? Extend DIRS to 8. That's the only change.)

Worked: Number of Islands

Grid of "1" (land) and "0" (water) — count the islands. This is connected components (Lesson 0028) wearing a costume: launch a flood from every unvisited land cell; each launch sinks one island:

def num_islands(grid):
    rows, cols = len(grid), len(grid[0])

    def sink(r, c):                                  # DFS flood fill
        if not (0 <= r < rows and 0 <= c < cols):    # off the map
            return
        if grid[r][c] != "1":                        # water, or already sunk
            return
        grid[r][c] = "0"                             # mark visited by sinking
        for dr, dc in [(-1,0),(1,0),(0,-1),(0,1)]:
            sink(r + dr, c + dc)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                sink(r, c)                           # one launch = one island
                count += 1
    return count

O(rows × cols) time — each cell is touched a constant number of times. Two idioms to note: marking in place (mutating the grid replaces the visited set — ask your interviewer if mutation is acceptable; keep a set if not), and checks at the top of the recursion (bounds/water/visited as guard clauses — cleaner than checking before each call, same prunes as Lesson 0027's Word Search).

Multi-source BFS: Rotting Oranges

Each minute, rotten oranges rot their fresh neighbors — how many minutes until all rot? "Simultaneous spread from many points, measured in time-steps" = BFS from all sources at once: seed the queue with every rotten orange, then Lesson 0020's level-size trick — each level is one minute:

q = deque(all initially rotten cells)      # multiple sources, one queue
minutes = 0
while q:
    for _ in range(len(q)):                # one level = one simultaneous minute
        r, c = q.popleft()
        for each fresh neighbor: rot it, enqueue it
    if q: minutes += 1

The tells: "spreads simultaneously," "minimum time/steps," "nearest X for every cell." DFS answers "connected?"; BFS answers "how far?" — same division of labor as trees, now with multiple starting points allowed.

Mission link · ML/AI

Flood fill segments images — the classic connected-components-on-pixels algorithm underlying mask tools and old-school vision pipelines. Multi-source BFS computes distance fields (every cell's distance to the nearest source), used in robotics path planning and game AI navigation meshes. And convolution's sliding neighborhood (Lesson 0007) meets its graph twin here: CNNs process exactly this cells-and-neighbors structure, learned instead of hand-coded.
