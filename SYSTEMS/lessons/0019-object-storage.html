Object Storage & Blobs

One skill, learned today: route large binary files around your database — into object storage, uploaded directly, served by CDN.

Why this matters

The moment an interview question mentions photos, videos, or files ("design Instagram/YouTube/Dropbox"), a specific three-part pattern is expected within a minute: object store + metadata in the DB + CDN in front. It's also lesson 0006's storage math coming due: text was 5 GB/day; images are 5 TB/day.

Why not the database?

Databases are engineered for many small rows with indexes, transactions, and constraints — none of which help a 4 MB JPEG. Stuffing blobs (binary large objects) into tables bloats backups, wrecks the buffer pool, and pays transactional overhead for bytes that never change. Object storage — S3, GCS, R2 — is the purpose-built home [AWS S3]:

PropertyWhat it means
Flat key → objecta giant KV store (lesson 0017!) for files: photos/42/beach.jpg → bytes. No folders, no queries — GET/PUT by key
Practically infinite, absurdly durableS3 advertises 99.999999999% durability — your object is replicated across facilities (0015, done for you)
Cheap & HTTP-native~$0.02/GB·month; every object has a URL — which means a CDN (0010) can cache it directly
Eventual-ish, no transactionsimmutable-write patterns (new key per version) sidestep consistency drama entirely

The pattern: metadata in the DB, bytes in the store, uploads direct

The database keeps a small row about the photo (owner, caption, timestamps, and the object key); the bytes never touch your API servers — clients upload straight to the object store using a presigned URL: a short-lived, signed link your server mints that grants "PUT exactly this key, for 10 minutes." AI-lab tie-in

Object storage is the bedrock of ML infrastructure: training datasets, model checkpoints (a frontier model's weights are a multi-hundred-GB "photo"), eval artifacts, batch inference outputs — all S3-shaped. The training-job pattern is exactly today's: metadata and lineage in a database, bytes in object storage, workers fetching by presigned URL.
