# Systems Design & Architecture Resources

## Knowledge

- [Book: _Designing Data-Intensive Applications_, 2nd ed. — Kleppmann & Riccomini (Dec 2025)](https://dataintensive.net/)
  The canonical depth book; 2nd edition rewritten for cloud-native and modern data systems. Use for: everything data-layer and distributed-systems (phases 2–3); read chapter-by-chapter once foundations are in place, not before.
- [Guide: System Design Primer — Donne Martin (GitHub, free)](https://github.com/donnemartin/system-design-primer)
  Most popular free system design compendium (290k+ stars), with Anki decks. Use for: topic index and second explanations of any concept; interview question bank.
- [Course: System Design in a Hurry — Hello Interview (free core)](https://www.hellointerview.com/learn/system-design/in-a-hurry/introduction)
  Written by former FAANG hiring managers; matches the actual interview format and rubric. Use for: interview framing, delivery structure, and mock-style practice in phase 5.
- [Platform: ByteByteGo — Alex Xu](https://bytebytego.com/)
  Visual case-study explanations; also covers ML/GenAI system design, which most prep material lacks. Use for: diagrams-first walkthroughs of real designs; AI-systems case studies late in the course.
- [Docs: MDN — How the web works / HTTP](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Web_standards/How_the_web_works)
  Precise, beginner-safe web fundamentals. Use for: phase-1 claims about HTTP, DNS, requests/responses — the primary source for early lessons.
- [Video: Gaurav Sen — System Design playlist (YouTube, free)](https://www.youtube.com/playlist?list=PLMCXHnjXnTnvo6alSjVkgxV-VH6EPyvoX)
  Low-friction video on-ramp for load balancing, queues, scaling tradeoffs. Use for: a second modality when a written explanation isn't landing.
- [Book: Site Reliability Engineering — Google (free online)](https://sre.google/sre-book/table-of-contents/)
  How production systems are actually run. Use for: phase 4 (SLOs, monitoring, incident response); skim-level, not cover-to-cover.
- [Challenges: Gossip Glomers — Fly.io + Kyle Kingsbury (free)](https://fly.io/dist-sys/)
  Hands-on distributed systems challenges (Maelstrom/Jepsen-based), doable in Python. Use for: phase-3 hands-on once queues/replication concepts exist. Too hard before that.

## Wisdom (Communities)

- [Hacker News](https://news.ycombinator.com/)
  Where real architecture writeups and postmortems get discussed by practitioners. Use for: reading discussions on engineering-blog posts linked from lessons; search hn.algolia.com for any system by name.
- [r/ExperiencedDevs](https://www.reddit.com/r/ExperiencedDevs/)
  Working engineers discussing real tradeoffs; well moderated. Use for: "how is this actually done in industry" questions once past the basics.
- [r/softwarearchitecture](https://www.reddit.com/r/softwarearchitecture/)
  Architecture-specific discussion. Use for: design critique on the capstone-style projects later in the course.
