Build a Real API

One skill, learned today: stand up a working HTTP API in Python and drive it with curl — the notes API from lesson 02, made real.

Why this matters

The mission says build, not just diagram. This little service is the seed of the capstone: over the coming lessons it gains a database (04), a container (32), and metrics (35). Every architecture concept lands harder when it happens to your service.

Setup (once)

FastAPI is the modern standard for Python APIs; uvicorn is the server program that listens on the port and hands requests to your code. Use a virtual environment so nothing pollutes your system Python:

mkdir -p ~/notes-api && cd ~/notes-api
python3 -m venv .venv && source .venv/bin/activate
pip install "fastapi[standard]"

The service

Save as main.py. Read it before you run it — every line maps to something from lesson 02:

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI()

notes: dict[int, dict] = {}          # in-memory "database" (this is a flaw. wait for it.)
next_id = 1

class NoteIn(BaseModel):             # the request-body contract
    text: str

@app.get("/notes")
def list_notes(limit: int = 20):
    return list(notes.values())[:limit]

@app.post("/notes", status_code=201)
def create_note(note: NoteIn):
    global next_id
    saved = {"id": next_id, "text": note.text}
    notes[next_id] = saved
    next_id += 1
    return saved

@app.get("/notes/{note_id}")
def get_note(note_id: int):
    if note_id not in notes:
        raise HTTPException(status_code=404, detail="note not found")
    return notes[note_id]

@app.delete("/notes/{note_id}", status_code=204)
def delete_note(note_id: int):
    notes.pop(note_id, None)         # idempotent: deleting twice is fine

Run it:

fastapi dev main.py                  # serves on http://127.0.0.1:8000

Drive it with curl

Second terminal. Predict each status code before you hit enter:

curl -i -X POST localhost:8000/notes -H "Content-Type: application/json" -d '{"text":"first note"}'
curl -i localhost:8000/notes/1        # expect 200
curl -i localhost:8000/notes/99       # expect 404
curl -i -X POST localhost:8000/notes -H "Content-Type: application/json" -d '{"wrong":"field"}'
curl -i -X DELETE localhost:8000/notes/1   # expect 204
curl -i -X DELETE localhost:8000/notes/1   # again! idempotent, still 204

That fourth call returns 422 — FastAPI validated the body against NoteIn and rejected it for you (422 is FastAPI's more specific flavor of 400-family "your request is wrong"). Now open localhost:8000/docs in a browser: FastAPI generated interactive API documentation from your code — a small taste of why API contracts matter [FastAPI tutorial].

Now break it

Create two notes. Stop the server (Ctrl-C). Start it again. curl localhost:8000/notes — empty. Your data lived in a Python dict, i.e. in process memory, and died with the process. Every crash, deploy, or restart wipes it — and a second server (lesson 07) would have a different dict. This exact flaw is why databases exist, and it's what lesson 04 fixes.
