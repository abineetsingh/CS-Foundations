Recursion II: Divide & Conquer

Today's skill: solve a problem by splitting it in half, solving both halves by trust, and merging — and read the complexity off the recursion tree.

Why this, now

Lesson 17's recursion shrank by one each call. Real power comes from shrinking by half and recursing on both halves. Merge sort is the canonical example — and the n log n you promised interviewers in Lesson 16 finally gets its proof.

Merge sort, via the three checks

def merge_sort(arr):
    if len(arr) <= 1:                    # 1. base: already sorted
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])         # 2. trust: halves come back sorted
    right = merge_sort(arr[mid:])
    return merge(left, right)            # 3. combine: merge two sorted lists

def merge(left, right):                  # two pointers (Lesson 06)!
    result, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])              # one side has leftovers
    result.extend(right[j:])
    return result

The leap of faith is doing serious work here: you cannot trace two branching calls per level — 2 + 4 + 8… frames. You verify base, trust, combine — done. Note merge is Lesson 06's two-pointer walk and Lesson 13's "merge two sorted lists," resurfacing as a subroutine.

Reading cost from the recursion tree

level 0:            [8 elements]              → merge work: 8
level 1:        [4]           [4]             → merge work: 4+4 = 8
level 2:     [2]   [2]     [2]   [2]          → merge work: 2×4 = 8
level 3:   [1][1] [1][1] [1][1] [1][1]        → base cases

Two observations, and the bound falls out:

  • Each level does O(n) total merge work — the pieces shrink but there are more of them; every element is merged once per level.
  • There are log₂n levels — halving until size 1 (Lesson 01's log intuition).

O(n) per level × O(log n) levels = O(n log n). This "work per level × number of levels" technique prices any divide & conquer — no formal recurrence-solving needed for interviews, though naming it (T(n) = 2T(n/2) + O(n)) earns points. Contrast: quicksort's partition can split badly (1 vs n−1), giving n levels → the O(n²) worst case from Lesson 16. Balance is where the log comes from.

Mission link · ML/AI

"Split, solve independently, merge" is the shape of parallel ML: data-parallel training splits the batch across GPUs and merges gradients (all-reduce is literally a merge tree with log-depth levels); map-reduce is D&C as infrastructure. The recursion-tree picture — total work per level × levels — is exactly how distributed-systems people budget communication costs.
