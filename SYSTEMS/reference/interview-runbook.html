Interview Runbook

The 45-minute algorithm on one page. Keep it beside you for every drill and mock. Print me.

1 · Requirements ~5 min

Functional (3–5, descope aloud): "I'll focus on X, Y — skipping Z, fair?"
Non-functional: scale (DAU), latency target, read:write ratio, consistency needs, availability.
Trap: starting to design before scoping. The descope IS the skill.

2 · Envelope ~5 min

Read QPS & write QPS (daily ÷ 10⁵; peak ×2–10). Storage (items × size × time × replicas). The ratio.
End with a verdict: "reads dominate 100:1 → cache-first; storage modest → defer sharding."
Trap: numbers that don't change the design.

3 · API + data model ~5 min

Core endpoints (REST nouns, pagination via cursor). Main tables/stores with keys; pick the store per access pattern.
Trap: gold-plating. The clock belongs to stage 5.

4 · High-level design ~10 min

Draw the boxes — client, DNS, CDN, LB, stateless fleet, cache, DB(+replicas), queue where async — while narrating the read path and write path end to end. A working v1, no cleverness yet.
Trap: a silent diagram. Talk through every arrow.

5 · Deep dives ~15 min — where you get hired

You propose the 2–3 hardest problems: the hot key, the fan-out, the consistency call, the failure mode. For each, options costed as "X because Y, alternative costs Z."
Trap: waiting to be led. Drive: "the interesting problem is the fan-out — shall I start there?"

Meta-skills (score continuously)

  • Drive. Announce structure, narrate transitions, propose deep dives yourself.
  • Tradeoffs out loud, always. Never a bare product name.
  • Declare budgets unprompted: staleness, latency, error budget.
  • Pushback = direction. "Great, that breaks my assumption — two mitigations…"
  • Seniority dial: junior → clean stage 4; senior → get to stage 5 fast, flag the hard parts first.

Panic list — if you blank

  1. Say the numbers. Envelope math buys thinking time and looks deliberate.
  2. Draw the v1 skeleton (glossary's phase-1 diagram). It's right often enough.
  3. Ask "what's the read path? the write path?" — narrating reveals the gaps.
  4. Name the classic hard part for this genre: hot key (feeds), ordering (chat), collisions (ids), fan-out (anything social).
  5. Reach for the patterns cheatsheet triggers.
