The Tree Problems Toolkit

Today's skill: solve the classic medium tree questions — diameter, balance, lowest common ancestor — by systematizing one meta-pattern.

Why this, now

This is a consolidation lesson: nothing new, everything combined. The pattern here — post-order, return what the parent needs — dissolves most medium tree problems on sight. That "on sight" feeling is your mission's pattern-recognition goal, achieved for one whole category.

The meta-pattern, stated once

  1. Decide what each subtree must report to its parent (a number, a boolean, a tuple).
  2. Write the base case for the empty tree.
  3. Recurse on both children (trust), combine, and — when the problem's answer isn't the root's report — update a nonlocal best as a side effect.

Step 3's distinction is the key insight of this lesson: the value flowing up the tree is often not the answer being sought.

Worked: Diameter (the archetype)

Longest path between any two nodes. The longest path through a given node = left height + right height. But the best path might be in a subtree, not through the root — so heights flow up, while the answer is collected on the side:

def diameter(root):
    best = 0
    def height(node):
        nonlocal best
        if not node:
            return 0
        left  = height(node.left)          # trust
        right = height(node.right)
        best = max(best, left + right)     # answer: path THROUGH this node
        return 1 + max(left, right)        # report: height, what parents need
    height(root)
    return best

One O(n) pass. Notice the two channels: return feeds the recursion; best collects the prize. "Longest path", "max path sum", "count good nodes" — all this exact shape.

Worked: Balanced? (enriching the report)

Is every node's subtree-height difference ≤ 1? The parent needs height and balance — so report both (here, −1 as a "broken" sentinel):

def is_balanced(root):
    def check(node):                  # returns height, or -1 if unbalanced below
        if not node: return 0
        left = check(node.left)
        right = check(node.right)
        if left == -1 or right == -1 or abs(left - right) > 1:
            return -1                 # short-circuit failure upward
        return 1 + max(left, right)
    return check(root) != -1

When the parent needs more than one fact, widen the report — a tuple or sentinel. Asking "what does the parent need to know?" first is what turns a blank stare into a plan.

Worked: Lowest Common Ancestor (reports meet)

def lca(node, p, q):
    if not node or node is p or node is q:
        return node                       # found one (or dead end)
    left  = lca(node.left,  p, q)
    right = lca(node.right, p, q)
    if left and right:                    # p and q split across my subtrees:
        return node                       #   I am the meeting point
    return left or right                  # both on one side (or neither)

Read it with the leap of faith: each side reports "the LCA if both targets are down here, else whichever target is down here, else None." The first node where both sides report something is where the paths converge. Three checks confirm it; tracing would take a page. (BST version: use the invariant — walk from the root until p and q split around you. O(h), no recursion needed.)

Mission link · ML/AI

"Aggregate child reports at each node" is message passing — the computational heart of graph neural networks, and of how MCTS backs value estimates up the game tree in AlphaZero-style agents. The two-channel trick (returned value ≠ collected answer) mirrors how backprop accumulates parameter gradients as a side effect while the chain rule's messages flow through.
