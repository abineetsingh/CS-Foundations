Sliding Window, Variable Size

Today's skill: find the longest (or shortest) contiguous stretch satisfying a condition — grow the right edge, shrink the left edge, never look back.

Why this, now

"Longest substring/subarray such that…" is one of the most common medium-difficulty question shapes. It's also your first taste of invariant-driven code — the style of reasoning that separates pattern-matching from actually understanding, and that interviews at research labs probe hard.

The shape

Unlike Lesson 0007's fixed k, here the window stretches and contracts to stay valid:

left = 0
for right in range(len(s)):        # grow: admit s[right]
    # ... update window state with s[right] ...
    while window_is_invalid():     # shrink until valid again
        # ... remove s[left] from window state ...
        left += 1
    best = max(best, right - left + 1)   # window [left..right] is valid

The invariant: after the inner while, the window is always valid. Since right advances n times and left only ever advances (at most n times total, across the whole run), the total work is O(n) — even though there's a loop inside a loop. This surprises everyone once: nesting isn't automatically O(n²) (Lesson 0001's rule 2 assumes the inner loop restarts; here it never does).

Worked example: Longest Substring Without Repeating Characters

def longest_unique(s):
    seen = set()          # chars currently in the window
    left = 0
    best = 0
    for right in range(len(s)):
        while s[right] in seen:      # invalid: duplicate incoming
            seen.remove(s[left])     # evict from the left
            left += 1
        seen.add(s[right])
        best = max(best, right - left + 1)
    return best

Trace "abcabcbb": the window stretches to abc, hits the second a, evicts the first a, and keeps sliding. Every character enters the set once and leaves at most once → O(n) time, O(min(n, alphabet)) space. Notice the toolkit stacking: a set (Lesson 0005) is the window summary (Lesson 0007) driven by two same-direction pointers (Lesson 0006).

Choosing your variant

Question shapeStrategy
Longest window satisfying a conditionGrow always; shrink only while invalid (code above)
Shortest window satisfying a conditionGrow until valid; then shrink while still valid, recording sizes
Count of valid windowsSame skeleton; add right − left + 1 per step
Mission link · ML/AI

"The longest recent stretch that fits a constraint" is exactly what a bounded context window with eviction does: new tokens enter on the right, old ones fall off the left when the budget is exceeded. Streaming-LLM/attention-sink papers are variable sliding windows with a twist about which tokens to evict.
