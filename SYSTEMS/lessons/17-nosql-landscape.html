The NoSQL Landscape

One skill, learned today: name the four NoSQL families, what each trades away, and answer "SQL or NoSQL?" like an engineer instead of a fan.

Why this matters

"Why did you pick this database?" is guaranteed interview material, and the wrong-store-for-the-job is a top cause of expensive rewrites in industry. The strong answer is never brand loyalty — it's matching access patterns to store shape, using everything phase 2 has taught you.

Why NoSQL exists at all

Relational databases were built machine-first: joins, constraints, and serializable transactions are easy when everything lives together, hard across shards. The 2000s web forced a choice, and NoSQL stores made it openly: give up joins and cross-row transactions; buy effortless horizontal scale — sharding and replication (15/16) built in as the default, not an operation you perform [DDIA ch. 2]. Everything else about them is downstream of that trade.

The four families

The recurring theme — NoSQL stores query by shape, not by question: you design the data around the exact access path (the wide-column partition IS the query; the document IS the page). Ad-hoc questions you didn't design for range from painful to impossible. SQL's superpower was always that it answers questions you hadn't thought of yet.

The decision framework

Walk it in order — out loud, in interviews:

The honest default

Start with Postgres. It handles JSON documents, full-text search, and — with the phase-2 toolkit of caching, replicas, and sharding — carries most businesses to enormous scale. Choose NoSQL for a named reason (a specific access pattern, write volume, or scaling requirement), and say the reason. "We used Mongo because it's webscale" is the answer interviewers are screening against.

AI-lab tie-in

A production agent stack quietly uses the whole landscape: Redis for session/state (KV), a document store for conversation transcripts, wide-column-ish event logs for telemetry, a graph for entity memory, Postgres for billing — plus vector indexes (45) as the newest family in the zoo. "Pick the store per access pattern" is literally the agent-infra job description.
