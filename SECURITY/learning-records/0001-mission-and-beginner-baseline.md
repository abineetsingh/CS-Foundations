# Mission established; security-beginner baseline

The SECURITY mission is set (2026-07-22): prepare for AppSec and security-engineering interviews while developing real, stack-agnostic secure-software ability. The user starts as a security beginner with Python, JavaScript, and web fundamentals and wants vulnerabilities taught through authorized practice, root-cause repair, and security-by-design prevention; secure architecture, practical cryptography, cloud/container hardening, and IAM follow the AppSec core.

**Implications:** Begin with adversarial thinking, trust boundaries, HTTP data flow, and safe testing rather than assuming security vocabulary. Use short visual lessons and tight-feedback exercises, keep all offensive work inside explicit training targets, and judge progress by threat models, code review, fixes, and verification—not by vulnerability-name recall.
