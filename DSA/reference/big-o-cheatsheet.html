The four reading rules

  1. Sequential blocks add — keep only the biggest. A loop (O(n)) followed by another loop (O(n)) is O(n), not "O(2n)".
  2. Nested blocks multiply. A loop inside a loop over the same n is O(n²).
  3. Drop constants. Doing 3 operations per element is still O(n).
  4. Drop lower-order terms. n² + n is O(n²) — for large n, the n² term dominates everything.

Tells: pattern → complexity

You see…It's usually
Direct index/key access, arithmetic, single comparisonO(1)
The problem size halves every iteration (n // 2, binary search)O(log n)
One loop over the inputO(n)
A sort, or a loop that does binary search each timeO(n log n)
Nested loops over the same input (all pairs)O(n²)
Recursion that branches twice per level without memoizationO(2ⁿ)

Interview phrasing

Say it like this: “This runs in O(n) time and O(1) extra space, where n is the length of the input array, because we make a single pass and keep only two variables.” Always name what n is, and always give space as well as time.
