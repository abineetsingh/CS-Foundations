Dynamic Programming II: Tabulation

Today's skill: flip top-down memoization into a bottom-up table — then shrink the table to O(1) space with rolling variables.

Why this, now

Same four questions, opposite direction: instead of recursing down from the goal and caching, fill answers up from the base cases. Interviewers often ask for both forms, then ask you to optimize space — this lesson is that exact sequence, rehearsed.

The flip

# top-down (Lesson 0034):                # bottom-up (today):
memo = {}                                dp = [0] * (n + 1)
def ways(i):                             dp[0], dp[1] = 1, 1        # bases first
    if i in memo: ...                    for i in range(2, n + 1):  # build upward
    ...recurse on i-1, i-2...                dp[i] = dp[i-1] + dp[i-2]
                                         return dp[n]

Identical state, transition, and bases — only the evaluation order changed: recursion discovers dependencies on demand; tabulation processes states in an order where dependencies are already filled (for 1-D problems: left to right). You've seen "process in dependency order" formalized before — it's topological sort (Lesson 0030); a DP's states form a DAG, and tabulation walks it topologically.

Worked: House Robber

Rob houses of values nums; adjacent houses can't both be robbed; maximize loot. Four questions: state = "best loot from the first i houses"; transition = at house i, either skip it (dp[i-1]) or rob it and skip its neighbor (dp[i-2] + nums[i]); bases = zero/one house; cache = the table:

def rob(nums):
    n = len(nums)
    if n == 1: return nums[0]
    dp = [0] * n
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    for i in range(2, n):
        dp[i] = max(dp[i-1],                # skip house i
                    dp[i-2] + nums[i])      # rob house i
    return dp[n-1]

O(n) time, O(n) space. The max is the only change from stair-counting — counting sums branches; optimizing maxes them. One template, two families of problems.

The space squeeze: rolling variables

Look at the transition: dp[i] reads only dp[i-1] and dp[i-2]. The rest of the table is dead weight — keep two variables:

def rob(nums):
    prev2, prev1 = 0, 0                 # best up to i-2, i-1
    for x in nums:
        prev2, prev1 = prev1, max(prev1, prev2 + x)
    return prev1

O(n) time, O(1) space — and this squeeze is mechanical: whenever the transition looks back a bounded distance, the table collapses to that many variables. (2-D tables similarly collapse to one or two rows — Lesson 0036.) Delivering the O(n)-table version, then squeezing it live when asked, is a rehearsed interview beat: rehearse it.

Mission link · ML/AI

An RNN is a rolling-variable DP: the hidden state carries exactly what the future needs from the entire past, in bounded space — prev1/prev2 generalized to a learned vector. And the memoization↔tabulation duality mirrors eager vs. graph execution in ML frameworks: same computation, different evaluation order, different systems trade-offs.
