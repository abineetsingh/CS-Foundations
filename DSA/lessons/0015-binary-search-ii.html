Binary Search II: Searching the Answer Space

Today's skill: binary search where there is no array at all — searching over possible answers.

Why this, now

This is the leap from "I know binary search" to "I see binary searches where others see nothing." It's a favorite medium/hard pattern at exactly the companies you're targeting, because it rewards modeling skill over memorization.

The reframe

Lesson 0014 searched an array for a value. But binary search really needs only two things:

  1. A range of candidate answers [lo, hi].
  2. A monotonic yes/no question: once the answer is "yes," it stays "yes" for everything beyond.
candidates:  3    4    5    6    7    8    9
feasible(x): no   no   no   yes  yes  yes  yes
                            ↑
                     binary search finds this boundary

Monotonicity plays the role sortedness played: it's what makes discarding half sound. An array of "no no no yes yes yes" is just a sorted array of booleans.

Worked example: Koko Eating Bananas

Koko has piles of bananas and h hours; eating speed k means each pile takes ⌈pile/k⌉ hours. Find the minimum k to finish in time.

Model it: candidates are speeds 1..max(piles). The question "can Koko finish at speed k?" is monotonic — faster never hurts. So:

import math

def min_eating_speed(piles, h):
    def feasible(k):                       # monotonic yes/no
        return sum(math.ceil(p / k) for p in piles) <= h

    lo, hi = 1, max(piles)                 # answer is guaranteed in here
    while lo < hi:
        mid = (lo + hi) // 2
        if feasible(mid):
            hi = mid                       # mid works — try smaller, keep mid
        else:
            lo = mid + 1                   # mid fails — everything ≤ mid fails
    return lo

Cost: O(n) per feasibility check × O(log(max_pile)) checks = O(n log M). Note this is the leftmost template from Lesson 0014 — "first k where feasible is yes" is literally bisect_left over an invisible boolean array.

The interview script: "The answer lies in [lo, hi]. Feasibility is monotonic because ___. So I'll binary search the answer and write an O(n) checker." Three sentences, and the hard problem is two easy ones.

Also in this family: rotated sorted arrays

"Search in Rotated Sorted Array" keeps the array but bends the rule: after computing mid, one half is always still sorted — decide which, check if the target's in it, discard accordingly. Same elimination logic, locally applied. Try it after the main problems.

Mission link · ML/AI

"Find the largest batch size that doesn't OOM" and "find the highest learning rate that doesn't diverge" are answer-space binary searches every ML engineer runs, sometimes by hand without noticing. Monotonic feasibility is also the shape of scaling-law sweeps: bigger compute budget, monotonically better loss — search the boundary you can afford.
