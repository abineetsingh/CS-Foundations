Binary Trees: Depth-First Search

Today's skill: traverse a binary tree recursively in all three orders, and know which order fits which job.

Why this, now

Trees are where your recursion investment pays out: a tree is a recursively defined structure (each child is the root of a smaller tree), so recursion isn't one technique among several — it's the native language. Tree questions are the most common interview category after arrays.

The structure

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left        # a smaller tree (or None)
        self.right = right      # a smaller tree (or None)

A linked list (Lesson 0012) is a tree where every node has one child. The upgrade to two children is what forces a choice — which child first? — and that choice is the whole topic.

Depth-first: go deep, then backtrack

DFS follows one branch to the bottom before trying siblings — it's just recursion's natural call order. The three variants differ only in when the node itself is handled — exactly the pre/post distinction from Lesson 0017's mystery function:

def preorder(node):                    # node, then children
    if not node: return
    visit(node)                        #   copy a tree, serialize
    preorder(node.left)
    preorder(node.right)

def inorder(node):                     # left, node, right
    if not node: return
    inorder(node.left)                 #   BSTs in sorted order (Lesson 0021)
    visit(node)
    inorder(node.right)

def postorder(node):                   # children, then node
    if not node: return
    postorder(node.left)               #   compute from subtree answers;
    postorder(node.right)              #   delete a tree safely
    visit(node)

All three: O(n) time (every node once), O(h) space for the call stack, where h is the height — log n if balanced, n if a degenerate chain. Saying "O(h), which is log n balanced, n worst case" is the complete interview answer.

The pattern that solves most tree problems

Ask each subtree for its answer, combine with the current node — post-order + the leap of faith:

def max_depth(node):
    if not node:
        return 0                       # base: empty tree has depth 0
    left = max_depth(node.left)        # trust
    right = max_depth(node.right)      # trust
    return 1 + max(left, right)        # combine

Swap the combine line and you get a different classic: left + right + node.val-style sums, min for shallowest leaf, tuple returns for balance checks. Lesson 0022 milks this systematically.

DFS without recursion — the stack, explicit

def preorder_iterative(root):
    stack = [root] if root else []
    while stack:
        node = stack.pop()
        visit(node)
        if node.right: stack.append(node.right)   # right first, so left pops first
        if node.left:  stack.append(node.left)

Same algorithm, same costs — you're managing Lesson 0010's stack yourself instead of borrowing Python's. Interviewers occasionally ask for this to check the recursion↔stack equivalence is real to you.

Mission link · ML/AI

Every JSON config, parse tree, and abstract syntax tree is walked with these exact traversals — a tokenizer's parse of your prompt included. Post-order is secretly the star: "compute children before parent" is the dependency order of a forward pass through a model's module tree, and its formal generalization (topological sort, Lesson 0030) schedules every computation graph in every framework.
