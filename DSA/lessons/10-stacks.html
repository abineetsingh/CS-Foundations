Stacks

Today's skill: recognize "most recent unfinished thing" problems and solve them with push/pop — including your first monotonic stack.

Why this, now

Stacks are the memory of nesting: brackets, function calls, undo histories. Beyond direct problems, the stack IS how recursion works (Lesson 17) — learn it here, and recursion later becomes "the stack Python manages for you."

LIFO, in five lines

A stack is Last-In-First-Out: you can only add to and remove from the top. In Python, a plain list is a perfect stack — both operations at the cheap end (Lesson 03):

stack = []
stack.append(x)     # push — amortized O(1)
top = stack[-1]     # peek — O(1)
x = stack.pop()     # pop  — O(1)
if not stack: ...   # empty check — O(1)

Never use insert(0)/pop(0) for a stack — that's the O(n) end.

Worked example: Valid Parentheses

Is "({[]})" properly nested? The insight: a closing bracket must match the most recent unmatched opener — "most recent unfinished thing" = stack.

def is_valid(s):
    pairs = {")": "(", "]": "[", "}": "{"}
    stack = []
    for ch in s:
        if ch in "([{":
            stack.append(ch)
        else:
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack        # leftovers = unclosed openers

O(n) time, O(n) space. Note both failure modes: closing with nothing open (not stack), and finishing with openers left (return not stack). Interviewers check both.

Level 2: the monotonic stack

Daily Temperatures: for each day, how many days until a warmer one? Brute force scans forward per day — O(n²). Instead, keep a stack of days still waiting for a warmer day. Each new temperature resolves every waiting day it beats:

def days_until_warmer(temps):
    result = [0] * len(temps)
    stack = []                          # indices; temps[stack] is decreasing
    for i, t in enumerate(temps):
        while stack and temps[stack[-1]] < t:
            j = stack.pop()             # day j finally sees a warmer day: i
            result[j] = i - j
        stack.append(i)
    return result

The stack stays sorted-descending automatically (monotonic). Each index is pushed once and popped at most once → O(n), by the same amortized argument as Lesson 08's window. The tell: "for each element, find the next greater/smaller element."

Mission link · ML/AI

Every deep learning framework holds a stack while your model runs: the call stack of nested module forward() calls, and autograd's tape unwinds in reverse (LIFO) order during backprop. When you read a stack trace in PyTorch, you are literally reading this data structure.
