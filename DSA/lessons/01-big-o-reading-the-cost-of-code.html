Big-O: Reading the Cost of Code

One skill, learned today: look at a short piece of Python and name how its cost grows as the input grows.

Why this, first

Every interview answer you ever give ends with the question “what's the complexity?” — at AI labs especially, where efficiency at scale is the whole business. Big-O is the lens through which all other DSA topics are seen, so it comes before any data structure.

What Big-O actually measures

Big-O does not measure seconds. It measures how the amount of work grows as the input grows. If you double the input:

  • O(1) — work stays the same (constant)
  • O(log n) — work grows by one step (logarithmic)
  • O(n) — work doubles (linear)
  • O(n log n) — work slightly more than doubles (the cost of sorting)
  • O(n²) — work quadruples (quadratic)

That's the entire vocabulary you need for now. These five classes cover the overwhelming majority of interview answers.

The four reading rules

To find a snippet's Big-O, apply these mechanically:

  1. Sequential blocks add — keep only the biggest. Two loops in a row is still O(n).
  2. Nested blocks multiply. A loop inside a loop over the same input is O(n·n) = O(n²).
  3. Drop constants. Three operations per element is still O(n) — growth is what matters, not the multiplier.
  4. Drop lower-order terms. n² + n is just O(n²); for n = 1,000,000 the n part is a rounding error.

Three worked examples

Example 1 — constant: O(1)

def first_item(arr):
    return arr[0]

Whether arr has ten elements or ten billion, this is one index lookup. Python lists are arrays underneath, so arr[0] jumps straight to the answer. O(1).

Example 2 — linear: O(n)

def total(arr):
    result = 0
    for x in arr:        # runs n times
        result += x      # O(1) work each time
    return result

One pass, constant work per element: n × O(1) = O(n). (The result = 0 line is O(1) — rule 1 says the loop dominates.)

Example 3 — quadratic: O(n²)

def has_duplicate(arr):
    for i in range(len(arr)):          # n times
        for j in range(i + 1, len(arr)):   # up to n times each
            if arr[i] == arr[j]:
                return True
    return False

A loop inside a loop, both over the input — rule 2 says multiply: O(n²). (The inner loop averages n/2 iterations, but rule 3 drops the ½.) In a few lessons you'll learn how a hash set turns this exact problem into O(n) — that upgrade is the single most common interview move.

Mission link · ML/AI

Transformer attention compares every token with every other token — the same all-pairs shape as Example 3. That's why attention is O(n²) in sequence length, why long context windows are expensive, and why so much research (FlashAttention, sparse attention, state-space models) attacks exactly this term. You can now read that sentence in any paper and know precisely what it means.
