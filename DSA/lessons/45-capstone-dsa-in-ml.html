The five ideas that kept recurring (the real curriculum)

  1. Cost is growth, not speed — and the recurring trades: space↔time (02), preprocess↔query (09), update↔recompute (07).
  2. Elimination needs justification — sortedness (06, 14), monotonicity (15), heap-minimality + non-negativity (32): every "discard half" carried a proof.
  3. Amortize over elements, not iterations — append (03), window (08), monotonic stack (10), rate limiter (44): the same proof, five costumes.
  4. Recursion = base + trust + combine — from fib to LCA to DP: induction, operationalized (17 → everywhere).
  5. Structures compose — map+list (43), two heaps (24), trie+DFS (25/27): requirements decompose, structures pair up.
