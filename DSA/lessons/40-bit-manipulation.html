Bit Manipulation (The Short Lesson)

Today's skill: the six bit operations, the four classic tricks — low frequency in interviews, but nearly free to learn.

Why this — honestly

Bit questions appear less often than everything else in this course, but the whole topic is one sitting, the tricks are memorable, and "Single Number" still shows up. Cost-benefit says: learn it once, lightly, now — then let the spaced-review quizzes maintain it.

The operations

a = 0b1100  (12)         AND  a & b   → 0b0100  (both bits set)
b = 0b0110  ( 6)         OR   a | b   → 0b1110  (either set)
                         XOR  a ^ b   → 0b1010  (exactly one set)
                         NOT  ~a      → -13     (Python: -(a+1), infinite-precision!)
                         SHIFT a << 2 → 48 (×4)   a >> 1 → 6 (÷2, floor)

bin(12) → '0b1100'       format(12, '08b') → '00001100'

Shifts ARE arithmetic: x << k = x·2ᵏ, x >> k = x // 2ᵏ — Lesson 14's halving, one instruction. One Python honesty note: ints are arbitrary-precision, so there's no 32-bit wraparound; problems that assume it ("reverse bits of a 32-bit integer") need explicit masking with & 0xFFFFFFFF.

The four tricks worth owning

1. XOR cancels pairs — Single Number

def single_number(nums):        # every element appears twice, except one
    result = 0
    for x in nums:
        result ^= x             # pairs annihilate: a ^ a = 0; 0 ^ x = x
    return result               # O(n) time, O(1) space

XOR is commutative and self-inverse, so the pairs vanish regardless of order. The hash-map solution (Lesson 05) costs O(n) space; XOR does it in O(1) — the follow-up interviewers fish for.

2. n & (n-1) clears the lowest set bit

def count_bits(n):              # how many 1-bits?
    count = 0
    while n:
        n &= n - 1              # one set bit vanishes per iteration
        count += 1
    return count                # loops once per SET bit, not per bit
# corollary: n & (n-1) == 0  ⇔  n is a power of two (or zero)

3. Check / set / toggle bit i

(n >> i) & 1        # read bit i
n | (1 << i)        # set bit i
n ^ (1 << i)        # flip bit i

4. Bitmask as a set — n booleans in one int

mask = 0b0101       # items {0, 2} chosen: integer AS a subset
# subsets of n items ↔ integers 0 .. 2ⁿ−1 — loop them:
for mask in range(1 << n):
    if mask & (1 << i): ...     # is item i in this subset?

This one earns its place: it re-expresses Lesson 26's subsets as counting, and powers "bitmask DP" (traveling-salesman-style states) — beyond interview scope, but now readable.

Mission link · ML/AI

Quantization — the technique that shrinks LLMs to run on laptops — is bit manipulation as a discipline: packing weights into 4-bit fields, unpacking with shifts and masks. Attention masks, GPU warp votes, and Bloom filters (hash-based set-membership with bit arrays — Lesson 05 meets today) all live at this level. You won't write it daily; you'll read it forever.
