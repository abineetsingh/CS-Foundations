Heaps & Priority Queues

Today's skill: maintain "the best element so far" under constant insertion — O(log n) per operation — and own the top-K pattern.

Why this, now

A heap answers one question relentlessly fast: what's the minimum right now? That single capability powers top-K problems, Dijkstra (Lesson 32), task schedulers — and beam search, the algorithm decoding every LLM response you've ever read.

The structure: a tree that lives in an array

A (min-)heap is a complete binary tree — every level full, last level packed left — with one rule: parent ≤ children. Weaker than the BST invariant (no left/right ordering — siblings are strangers), and that weakness is the point: less order costs less to maintain.

Completeness means no pointers are needed — the tree packs into an array (Lesson 03), read level by level:

heap = [1, 3, 2, 8, 5, 4]         1          parent of i:  (i − 1) // 2
                                 / \         children of i: 2i + 1, 2i + 2
index:  0  1  2  3  4  5        3   2
                               / \  /        min = heap[0], always. O(1) peek.
                              8  5 4

The two repairs

  • push: append at the end (keeps completeness), then sift up — swap with parent while smaller. Path length ≤ height = O(log n) (complete trees are always balanced — the log is guaranteed, unlike BSTs).
  • pop-min: take heap[0], move the last element to the root, then sift down — swap with the smaller child while larger. O(log n).

Python: heapq

import heapq

h = []
heapq.heappush(h, 5)          # O(log n)
heapq.heappush(h, 1)
smallest = h[0]               # O(1) peek
x = heapq.heappop(h)          # O(log n) → 1

heapq.heapify(arr)            # existing list → heap, O(n) (!in place, faster than n pushes)

# heapq is MIN-only. Max-heap: negate on the way in and out.
heapq.heappush(h, -score)
best = -heapq.heappop(h)

# Non-numbers: push tuples — compared elementwise (Lesson 16)
heapq.heappush(h, (priority, task_id, task))

The signature pattern: top-K

Find the k largest of n elements. Full sort: O(n log n). Better: keep a min-heap of size k — the doorman only needs to know the weakest current member:

def k_largest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)                  # O(k)
    for x in nums[k:]:
        if x > heap[0]:                  # beats the weakest of the elite?
            heapq.heapreplace(heap, x)   # evict weakest, admit x — O(log k)
    return heap

O(n log k) time, O(k) space. The counterintuitive detail interviewers probe: k largest uses a min-heap — the root is the eviction candidate. Streaming-friendly too: n never needs to fit in memory. (Also know: heapq.nlargest(k, nums) does this in one call; "kth largest" has an O(n)-average quickselect alternative — name it, offer the heap.)

Mission link · ML/AI

Beam search is top-K as a decoding strategy: at each step, expand candidate sequences, score them, and keep only the k best in exactly this heap pattern — BFS (Lesson 20) with a bouncer. Top-k sampling does it over logits every single token. And vector-database retrieval returns "k nearest neighbors" maintained with — a size-k heap over similarity scores. This lesson is running in production every time you talk to a model.
