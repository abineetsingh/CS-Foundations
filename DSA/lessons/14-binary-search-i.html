Binary Search I: The Sorted-Array Classic

Today's skill: write a binary search that is correct the first time — using an invariant, not luck.

Why this, now

Binary search is the canonical O(log n) algorithm — the payoff of Lesson 01's "halving" intuition. It's also famously easy to get subtly wrong: the first published binary search took years to debug, and most candidates still fumble the boundaries. Writing it invariant-first is a visible skill signal.

The idea

In a sorted array, comparing the target to the middle element eliminates half the candidates — the same "provable elimination" logic as two pointers (Lesson 06), but discarding half instead of one. Halving n until 1 takes log₂n steps: O(log n). A billion elements → ~30 comparisons.

The template — and the invariant that makes it correct

def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1          # candidates live in [lo, hi]
    while lo <= hi:                   # inclusive range → <=
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1              # mid is ruled out → step PAST it
        else:
            hi = mid - 1              # mid is ruled out → step PAST it
    return -1                          # range empty: not present

The invariant: if the target exists, it is within [lo, hi]. Every line serves it:

  • lo <= hi: the range [lo, hi] is only empty when lo > hi — stop exactly then.
  • mid ± 1: arr[mid] was examined and ruled out, so exclude it — keeping it (e.g. lo = mid) can loop forever when the range narrows to one element.
  • Progress: each iteration strictly shrinks the range → termination is guaranteed.

When you're unsure about a boundary, don't pattern-match — ask "does this line preserve the invariant?" That question repairs any variant you'll ever write.

The variant you actually need: leftmost position

Interviews rarely want "any match" — they want the first position where the target could go (handles duplicates, absent targets, insertion points):

def leftmost(arr, target):            # first index i with arr[i] >= target
    lo, hi = 0, len(arr)              # note: hi = n, range is [lo, hi)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid                  # arr[mid] >= target: mid stays a candidate
    return lo

This is exactly Python's bisect.bisect_left — know the module exists, and be able to write it anyway.

Mission link · ML/AI

Remember Lesson 09's cumulative distribution? Sampling a token = drawing a random number and binary-searching the CDF for where it lands — bisect over prefix sums. Also everywhere in ML tooling: hyperparameter bisection, finding the largest batch size that fits in memory (binary search on "does it OOM?"), git bisect for regressions.
