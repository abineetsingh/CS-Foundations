Search

One skill, learned today: explain why databases can't search text, how an inverted index can, and how a search system stays fed.

Why this matters

"Add search" appears in half of all design questions (search tweets, products, videos…) and has a signature wrong answer — LIKE '%query%' — that interviewers wait for you to avoid. The right answer introduces your first derived data system, a pattern that phase 3 generalizes.

Why the database fails at this

A B-tree index (0013) needs a sorted prefix to seek into. WHERE text LIKE 'compil%' can use it; WHERE text LIKE '%compiler%' cannot — a match can start anywhere, so the database full-scans every row, O(n) at query time. Ranking ("best match first") makes it worse: SQL has no notion of relevance. Text search needs a different index shape entirely.

The inverted index

Flip the mapping: instead of document → words, store word → documents containing it (the "posting list"). Your DSA hash-map instincts (0005) apply directly [Elasticsearch: index basics]:

Documents:                          Inverted index:
  d1: "the fast compiler"             fast     → [d1, d3]
  d2: "machines that think"           compiler → [d1, d3]
  d3: "a fast small compiler"         machine  → [d2]      ← note: normalized!
                                      think    → [d2]
                                      small    → [d3]

Query "fast compiler" → lookup both words → intersect [d1,d3] ∩ [d1,d3] → rank → done.
O(query words), not O(all documents).

Building the index does the linguistic work once, at write time — tokenize (split words, drop "the"), normalize ("Machines" → "machine", stemming) — so queries are hash lookups plus list intersections. Ranking classics: TF-IDF/BM25 — score documents up for containing rare query words often, down for being long. That's enough ranking vocabulary for interviews.

AI-lab tie-in

RAG is this lesson with embeddings: BM25 finds documents sharing your words; vector search (0045) finds documents sharing your meaning — an "inverted index" over semantic space using ANN structures (0013's tie-in). Production retrieval is usually hybrid: BM25 + vectors, scores fused. Master today's half and the modern half is one new index away.
